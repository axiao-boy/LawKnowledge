# 添加文档指南

本文档说明如何在 LawKnowledge 项目中手动添加新文档。

## 📝 手动添加文档需要修改的位置

在当前的 VitePress 项目中，手动添加一篇新文档需要修改 **2-3 个位置**：

---

### 1️⃣ 创建 Markdown 文件（必须）

在 `docs/` 目录下创建新的 `.md` 文件，路径决定 URL。

**示例**：
```
docs/labor-law/new-topic.md   →  URL: /labor-law/new-topic
```

---

### 2️⃣ 更新侧边栏（必须）

修改 `docs/.vitepress/config.mts` 中的 `sidebar` 配置：

```typescript
sidebar: {
  '/': [
    {
      text: '劳动法和劳动合同法',
      items: [
        { text: '劳动法', link: '/labor-law/labor' },
        { text: '劳动合同法', link: '/labor-law/contract' },
        { text: '你的新文档标题', link: '/labor-law/new-topic' }  // ← 添加这一行
      ]
    }
  ]
}
```

---

### 3️⃣ 更新顶部导航（可选）

如果新文档是一个**独立大分类**，需要修改 `nav` 配置：

```typescript
nav: [
  { text: '首页', link: '/' },
  { text: '劳动法和劳动合同法', link: '/labor-law/' },
  { text: '新分类', link: '/new-category/' }  // ← 添加新导航项
]
```

---

### 4️⃣ 更新首页（可选）

如果想在首页展示新文档入口，修改 `docs/index.md`：

```markdown
- [新文档](/labor-law/new-topic) - 文档描述
```

---

## 📋 添加文档流程图

```
创建 .md 文件
    ↓
更新 config.mts 的 sidebar（关键）
    ↓
[可选] 更新 config.mts 的 nav（如果是新分类）
    ↓
[可选] 更新 index.md（首页入口）
    ↓
开发服务器自动热更新，刷新页面即可看到
```

## 💡 路径规则

| Markdown 文件路径 | 访问 URL |
|-------------------|----------|
| `docs/index.md` | `/` |
| `docs/labor-law/index.md` | `/labor-law/` |
| `docs/labor-law/labor.md` | `/labor-law/labor` |

**注意**：link 路径不要带 `.md` 后缀，直接写目录/文件名即可。
